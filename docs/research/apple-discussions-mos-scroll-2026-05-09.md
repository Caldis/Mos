# Apple Support Communities: Mos / smooth scrolling scan

Date: 2026-05-09
Scope: discussions.apple.com only. GitHub issues excluded from evidence.

## Relevant Threads Analyzed

1. https://discussions.apple.com/thread/255899023
   - Direct Mos mention. User installed Mos to improve a Microsoft Surface Arc Mouse on an M1 Mac; after uninstalling, Magic Mouse gestures and scrolling did not work. User later reported macOS 15.2 fixed it.
   - Constructive point: uninstall / interaction troubleshooting and clearer recovery guidance.

2. https://discussions.apple.com/thread/252108130
   - Direct Mos mention. User could not scroll YouTube pages in Safari 14 after Big Sur 11.0.1; later said the cause was a third-party app, Mos. The linked GitHub issue was not used as evidence.
   - Constructive point: Safari / YouTube compatibility regression and app exclusion behavior.

3. https://discussions.apple.com/thread/252336951
   - Direct Mos mention plus exact problem. Users report choppy scrolling on M1 Macs; one user recommends Mos for non-Apple mice, while others discuss Bluetooth, Safari, Logitech, inertia settings, Feedback reports, and OS updates.
   - Constructive point: non-Apple mouse smooth scrolling demand; per-device/browser/OS compatibility.

4. https://discussions.apple.com/thread/251304067
   - Exact problem. User says macOS smooth scrolling works for Magic Mouse / Magic Trackpad but not third-party ergonomic or gaming mice, including Logitech, and asks how to enable it system-wide.
   - Constructive point: system-wide smooth scrolling for any pointing device.

5. https://discussions.apple.com/thread/7678111
   - Exact problem. User reports Chrome and Safari scrolling became less smooth after Sierra; a third-party app developer recommends an app that animates scroll and adds non-Apple mouse controls.
   - Constructive point: OS-update induced browser smoothness regression; third-party utility demand.

6. https://discussions.apple.com/thread/762414
   - Exact problem. User asks whether continuous smooth scrolling is possible on Mac; reports line-by-line scrolling and asks whether a special mouse is required.
   - Constructive point: need to transform discrete wheel ticks into continuous movement.

7. https://discussions.apple.com/thread/1409141
   - Exact problem. User argues scrolling should accelerate/decelerate naturally across apps and complains jump scrolling makes orientation difficult.
   - Constructive point: human-perceived motion, inertia, and orientation-preserving scroll behavior.

8. https://discussions.apple.com/thread/5469702
   - Adjacent problem. Users report jerky/slower scrolling after Mavericks in browsers and other apps; suggestions include inertia settings and browser comparisons, with mixed success.
   - Constructive point: diagnostics and fallback toggles for inertia / responsive scrolling regressions.

9. https://discussions.apple.com/thread/255838134
   - Adjacent problem. User reports choppy macOS Sequoia scrolling in System Settings and Notification Center and asks whether it is a known bug or OS-version issue.
   - Constructive point: detect and communicate OS-level limitations outside Mos control.

10. https://discussions.apple.com/thread/255811597
    - Adjacent problem. User reports Safari / YouTube scrolling lag on M1 MacBook Air and asks if it is software or hardware; replies suggest OS updates and malware checks.
    - Constructive point: troubleshooting flow for app-specific scroll lag.

11. https://discussions.apple.com/thread/254164793
    - Adjacent problem. User reports persistent UI and browser scroll stutter across Music, App Store, Safari, Chromium browsers, and even demo devices.
    - Constructive point: broader scroll-performance pain exists beyond mouse-wheel smoothing.

## Count

At least 7 constructive Mos-relevant points were found if counting only direct Mos and exact smooth-scroll / third-party-mouse problem threads. Including adjacent OS/browser stutter diagnostics, 11 relevant threads were analyzed.

## Topic Groups

- Third-party mouse smooth scrolling gap: non-Apple mice lack Magic Mouse / Trackpad-like smoothness or inertia.
- Mos-specific compatibility and recovery: Safari/YouTube scroll blocking; Magic Mouse gesture/scroll confusion after Mos install/uninstall.
- OS/browser regressions: Sierra, Mavericks, Big Sur/M1, Sequoia reports.
- Hardware / connection variables: Logitech mice, Microsoft Arc Mouse, Bluetooth, Unifying receiver, Magic Mouse, trackpad comparisons.
- User expectations: continuous movement, acceleration/deceleration, lower cognitive load, system-wide consistency.

## Sentiment

Mixed but useful. Demand-side sentiment is strongly frustrated and problem-aware; Mos appears as both a recommended free workaround and, in two threads, a possible contributor to compatibility trouble.

## Implications

- Product: prioritize per-app exclusions, Safari/YouTube compatibility, uninstall/reset guidance, and clearer compatibility notes for Magic Mouse vs third-party mice.
- Support: publish a short troubleshooting page for "scroll stopped / Safari won't scroll / Magic Mouse gestures missing after Mos".
- Business: Apple Communities show organic demand for the core Mos value proposition, but trust depends on avoiding or clearly recovering from edge-case regressions.

## Effective

Yes. Apple Support Communities contains more than 3 constructive suggestions/questions/problems involving Mos directly or the exact user problem Mos addresses.
